## 1. 测试概述

### 1.1 测试目标

本测试方案针对 NeuroCashEngine v1.0.0 MVP 产品，旨在：

1. **功能验证**：确保所有 MVP 功能按 PRD 要求正常工作
2. **需求覆盖**：实现 PRD 需求点 100% 覆盖
3. **用户体验**：验证产品易用性和用户友好度
4. **质量保证**：确保产品达到金融级别的安全性和稳定性

### 1.2 测试范围

| 模块 | 包含功能 | 优先级 |
|------|---------|-------|
| **M1 - 用户账户体系** | 用户注册（个人/企业）、登录、账户管理、行业分类选择 | P0 |
| **M2 - C端基础功能** | 资产账户管理、资产概览、官方/非官方资讯 | P0 |
| **M3 - B端现金流管理** | 多账户聚合、现金流预测与预警、资金调拨 SOP 生成 | P0 |
| **M4 - 数据产品 API** | C端投资情绪指数 API、数据脱敏与安全 | P0 |

### 1.3 需求覆盖矩阵

本方案完全覆盖 PRD 8.1 节定义的 MVP 范围：

| PRD 需求点 | 测试用例ID | 优先级 | 状态 |

## 7. 缺陷管理

### 7.1 缺陷等级定义

| 等级 | 描述 | 响应时间 |
|------|------|---------|
| **P0 - 严重/阻塞** | 系统崩溃、核心功能不可用、数据丢失 | 立即修复 |
| **P1 - 重要** | 主要功能异常，但有替代方案 | 4小时内响应 |
| **P2 - 一般** | 次要功能问题，不影响主要流程 | 24小时内响应 |
| **P3 - 轻微** | UI问题、建议类、文案错误 | 下一版本修复 |

### 7.2 缺陷管理流程

1. 发现缺陷 → 记录缺陷（包含复现步骤、截图、环境信息）
2. 评估与定级 → 确定优先级和严重程度
3. 分配与修复 → 开发人员修复
4. 验证与关闭 → 测试人员验证修复后关闭
5. 回归测试 → 在修复后进行回归
