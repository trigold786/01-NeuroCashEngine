## 1. 验收范围与目标

### 1.1 验收目标

本次 UAT 旨在验证 NeuroCashEngine v1.0.0 MVP 是否按照 BRD v0.2.0 和 PRD v1.0.0 正确实现业务需求，前后端数据一致，功能可用好用，并叠加中国合规（等保2.0 / PIPL）要求进行验收。

### 1.2 验收功能域

| 功能域 | 子模块 | 优先级 | BRD追溯 | PRD追溯 |
|--------|--------|--------|---------|---------|
| **用户账户体系** | 注册(个人/企业)、登录、JWT认证、RBAC权限 | P0 | §3.1 | §2.1 |
| **C端 - 资产概览** | 四大产品投资比例展示(饼图/柱状图)、数据脱敏切换 | P0 | §3.1.1 | §3.1.1 |
| **C端 - 官方资讯** | 资讯列表/详情/搜索/分类过滤/来源标签 | P0 | §3.1.1 | §3.1.2 |
| **C端 - 投资策略** | 风险问卷、MPT推荐、产品列表、模拟回测 | P0 | §3.1.2 | §3.2.2 |
| **B端 - 现金流管理** | 多账户聚合、预测预警、事件标注、SOP生成/导出 | P0 | §3.2.1 | §4.1.1-4.1.3 |
| **B端 - 投资管理** | 企业风险评估、产品推荐、组合监控/再平衡 | P0 | §3.2.2 | §4.2.1-4.2.3 |
| **数据产品API** | 5个脱敏API、OAuth2.0/API Key认证、限流、文档 | P0 | §3.3 | §5.1-5.3 |
| **商业化** | 三级订阅、积分体系、推荐返佣 | P0 | §4 | §6 |
| **非功能性** | 安全性(JWT/RBAC/加密)、TS编译、容器化部署 | P0 | — | §7 |
| **交互原型** | 18个页面路由、暗黑/浅色双主题、财富金色强调、等宽字体 | P0 | — | §9.1-9.2 |

### 1.3 验收范围说明

- **纳入验收**：Web 前端 PWA + 后端微服务集群（account-center / cash-flow-engine / content-hub / data-product-api）+ Docker 容器化部署
- **不纳入验收**：iOS/Android 原生 APP（工程预留但未实现）、微信/支付宝小程序（未实现）、第三方 OAuth 银行授权（未实现）、HTTPS 传输层（需生产配置）、性能压力测试（需专用环境）
